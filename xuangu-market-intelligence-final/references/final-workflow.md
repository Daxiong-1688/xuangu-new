# 最终工作流

## 目标

将用户最终确认的研究过程直接执行，不重演中间UI和内容调优。

```text
最新完成交易日
→ 全市场真实行情与证据
→ 标准化与完整性校验
→ 动态识别领涨/领跌/转强/异常板块
→ 板块涨跌六维归因
→ 市场状态与风险指数
→ 多周期条件预测
→ 产业链和公司筛选
→ 三层股票池与风险门槛
→ 固定黄金HTML渲染
→ 源码、运行时DOM和视觉验证
```

## 执行职责

- 程序负责：Yixin调用、原始响应、标准化、横截面计算、保守基准Bundle、事实保护、黄金渲染和验收。
- AI负责：读取本次证据后的板块因果归因、事件核验、条件预测和公司级研究补丁。
- AI不得直接修改事实层或HTML；程序不得把启发式归因伪装成已验证催化。

## HTML语义验收

- 涨跌数值与红绿语义一致。
- 置信度、评分、进度条与生命周期位置一致。
- 未来7天标题、日期卡片和事件分布一致；无事件日期显示明确监测占位。
- 预测排行只允许出现本次Bundle的动态板块。
- 候选、重点、核心按钮进入文字对应层级。
- 非 `evidence_constrained_ai` Bundle在页面顶部显示草稿状态。
